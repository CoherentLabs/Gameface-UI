title: Switch

The `Switch` component is a versatile UI element that allows users to toggle between two states, such as "on" and "off."

## Usage

To use the `Switch` component, you can customize its labels and behavior as shown below:

```tsx
import Switch from '@components/Basic/Switch/Switch';

const App = () => {
    return (
        <Switch>
            <Switch.LabelLeft>Off</Switch.LabelLeft>
            <Switch.LabelRight>On</Switch.LabelRight>
        </Switch>
    );
};

export default App;
```

## API

### Props

| Prop Name       | Type                        | Default     | Description                                                                                                   |
|------------------|-----------------------------|-------------|---------------------------------------------------------------------------------------------------------------|
| `style`         | `JSX.CSSProperties`        | `{}`        | Inline styles to apply directly to the root element of the component.                                        |
| `class`         | `string`                   | `""`        | Additional CSS classes to apply to the component.                                                            |
| `ref`           | `SwitchRef \| undefined`   | `undefined` | A reference to the component, providing access to its methods and underlying HTML element.                   |
| `disabled`      | `boolean`                  | `false`     | Disables the switch, preventing user interaction.                                                            |
| `class-checked` | `string`                   | `""`        | Additional CSS classes applied when the switch is checked.                                                   |
| `class-disabled`| `string`                   | `""`        | Additional CSS classes applied when the switch is disabled.                                                  |
| `checked`       | `boolean`                  | `false`     | Specifies the initial checked state of the switch.                                                           |
| `onChange`      | `(checked: boolean) => void`| `undefined` | Callback function triggered whenever the switch is toggled, providing the current checked state.             |

### Methods

| Method       | Parameters         | Return Value | Description                                                                 |
|--------------|--------------------|--------------|-----------------------------------------------------------------------------|
| `setChecked` | `Setter<boolean>`  | `void`       | Programmatically sets the checked state of the switch based on the provided boolean value. |

## Slots

### `Switch.LabelLeft`

This slot enables customization of the left label of the switch. You can use it to include additional elements or modify the content of the left label.

#### Properties

| Prop Name | Type         | Default     | Description                                                                 |
|-----------|--------------|-------------|-----------------------------------------------------------------------------|
| `children`| `JSX.Element`| `undefined` | The content to display inside the left label, which can be any valid JSX element. |

#### Example

```tsx
import Switch from '@components/Basic/Switch/Switch';

const App = () => {
    return (
        <Switch>
            <Switch.LabelLeft>Off</Switch.LabelLeft>
        </Switch>
    );
};

export default App;
```

### `Switch.Control`

This slot allows you to customize the appearance and behavior of the `Switch` control when it is in its unchecked state. You can use it to apply custom styles, add CSS classes, or include additional elements within the control.

#### Properties

| Prop Name | Type               | Default | Description                                                                 |
|-----------|--------------------|---------|-----------------------------------------------------------------------------|
| `style`   | `JSX.CSSProperties`| `{}`    | Inline styles to apply directly to the control element.                     |
| `class`   | `string`           | `""`    | Additional CSS classes to style the control element.                        |
| `children`| `JSX.Element`      | `undefined` | Content to render inside the control, which can be any valid JSX element. |

#### Example

```tsx
import Switch from '@components/Basic/Switch/Switch';

const App = () => {
    return (
        <Switch>
            <Switch.LabelLeft>Off</Switch.LabelLeft>
            <Switch.LabelRight>On</Switch.LabelRight>
            <Switch.Control style={{ backgroundColor: 'gray' }}>
            </Switch.Control>
        </Switch>
    );
};

export default App;
```

### `Switch.Indicator`

This slot allows you to customize the indicator displayed when the switch is toggled. It can be styled or replaced with custom elements to suit your design needs.

:::note
The `Switch.Indicator` slot must be used within the `Switch.Control` slot.
:::

#### Properties

| Prop Name | Type               | Default | Description                                                                 |
|-----------|--------------------|---------|-----------------------------------------------------------------------------|
| `style`   | `JSX.CSSProperties`| `{}`    | Inline styles to apply directly to the indicator element.                   |
| `class`   | `string`           | `""`    | Additional CSS classes to style the indicator element.                      |
| `children`| `JSX.Element`      | `undefined` | Content to render inside the indicator, which can be any valid JSX element. |

#### Example

```tsx
import Switch from '@components/Basic/Switch/Switch';

const App = () => {
    return (
        <Switch>
            <Switch.LabelLeft>Off</Switch.LabelLeft>

```

### `Switch.Handle`

This slot allows you to customize the handle of the `Switch` component. You can use it to apply custom styles, add CSS classes, or include additional elements.

:::note
The `Switch.Handle` slot must be used within the `Switch.Control` slot.
:::

#### Properties

| Prop Name       | Type               | Default | Description                                                                 |
|------------------|--------------------|---------|-----------------------------------------------------------------------------|
| `style`         | `JSX.CSSProperties`| `{}`    | Inline styles to apply directly to the handle element.                      |
| `style-checked` | `JSX.CSSProperties`| `{}`    | Inline styles to apply when the switch is in the checked state.             |
| `class`         | `string`           | `""`    | Additional CSS classes to style the handle element.                         |
| `class-checked` | `string`           | `""`    | Additional CSS classes to style the handle when the switch is checked.      |
| `children`      | `JSX.Element`      | `undefined` | Content to render inside the handle, which can be any valid JSX element.    |

:::note
The `style` and `class` properties are applied when the switch is unchecked, while `style-checked` and `class-checked` are applied when it is checked.
:::

#### Example

```tsx
import Switch from '@components/Basic/Switch/Switch';

const App = () => {
    return (
        <Switch>
            <Switch.LabelLeft>Off</Switch.LabelLeft>
            <Switch.LabelRight>On</Switch.LabelRight>
            <Switch.Control style={{ backgroundColor: 'gray' }}>
                <Switch.Indicator style={{ background: 'white' }}></Switch.Indicator>
                <Switch.Handle style={{ background: 'red' }} style-checked={{ background: 'black' }}></Switch.Handle>
            </Switch.Control>
        </Switch>
    );
};

export default App;
```

## Guide

### Set the value of the Switch Programmatically 

The `Switch` component lets you set the `checked` state through code.

#### Steps

1. Declare a `ref` variable with type `SwitchRef` and pass it to the `Switch` component.
2. Call `ref.setChecked()` from anywhere (e.g., a keyboard event or external button) to set the checked state programmatically.

```tsx
import Switch, { SwitchRef } from '@components/Basic/Switch/Switch';

const App = () => {
    let switchRef!: SwitchRef
    const handleKeyPress = (e: KeyboardEvent) => {
        if (e.keyCode !== 13) return
        
        switchRef.setChecked(true);
    }

    return (
        <>
            <Switch ref={switchRef} keypress={handleKeyPress}></Switch>
        </>
    );
};

export default App;
```
