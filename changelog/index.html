Changelog

3.0.1 Latest

Section titled “3.0.1 ”Latest

🛠 Navigation Improvements

Section titled “🛠 Navigation Improvements”

  • Fix Navigation: callback in the ActionCfg type is now optional. Overriding a default action with only key/button bindings no longer requires passing an empty callback or a redundant global flag.

  • Feature Navigation: addAction now warns at runtime when the provided config has no callback and no global: true, or no key/button binding, as such an action would be unreachable or produce no effect.

3.0.0

Section titled “3.0.0”

⚠️ Breaking Changes

Section titled “⚠️ Breaking Changes”

  • Breaking Change Removed default width and height styles from Relative component to avoid unexpected behavior. Now, the Relative component will only take up as much space as its content requires unless dimensions are explicitly set through styles or classes.

  • Breaking Change Removed the <div> wrapper from the Tab component to avoid unexpected layout behavior. Now, the Tab component will render its content directly without an additional wrapper element. This change may affect styling and layout, so please review your usage of the Tab component and adjust styles accordingly.

✨ New View UI

Section titled “✨ New View UI”

  • New Added a new sample Inventory UI prototype view at src/views/inventory. It can be accessed by navigating to /inventory in the development server.

  • New Added new assets related to the Inventory UI (icons & font).

    Asset Credits

    The “Cinzel” font is provided by Google Fonts under the SIL Open Font License. The UI icons are provided by game-icons.net under the CC BY 3.0 license.

🛠 Component Enhancements

Section titled “🛠 Component Enhancements”

  • Feature Added center prop to the Absolute component for easily centering content.

  • Feature Added dot-notation access for Column sizes. This allows you to use specific Column sizes without needing to import each one separately. For example, you can now write <Column.Six /> instead of <Column6 />.

    Caution

    The previous method of importing individual size components is still supported for backward compatibility. You can continue using <Column1>, <Column2>, etc., as before.

  • Feature Added fill prop to the Icon & Image components allowing them to fill the space of their parent containers.

  • Fix Icon: Fixed typescript types suggestion for the component’s props.

  • Feature Added width and height props to the Block component for easier size customization when prototyping.

  • Feature Added dot-notation access for the Top, Content, and Bottom components within the Layout component. You can now write <Layout.Top>, <Layout.Content>, and <Layout.Bottom> without needing to import them separately.

    Caution

    The previous method of importing these components individually is still supported for backward compatibility. You can continue using <Top>, <Content>, and <Bottom> as before.

2.1.0

Section titled “2.1.0”

  • Feature Added Grid.Tile slot for the Grid. This allows you to use the GridTile component without importing it separately. You can now write <Grid.Tile row={1} col={1}> instead of <GridTile row={1} col={1}>.

    Caution

    The previous method of importing GridTile separately is still supported for backward compatibility. You can continue using <GridTile> as before if you prefer.

  • Feature Added gap, row-gap, and column-gap props to the Flex & Grid components.

    Caution

    These properties are supported in Gameface 2.2 and later versions.

  • Feature Added row-class, column-class, row-style, and column-style props to the Grid component for customizing the styles of internal row and column wrappers.

  • Fix Dropdown: Fixed content inside Dropdown.Trigger not being vertically centered.

2.0.0

Section titled “2.0.0”

⚠️ Breaking Changes

Section titled “⚠️ Breaking Changes”
  • Breaking Change The ref prop on layout components (Absolute, Flex, Block, etc.) that don’t have any other methods or properties now exposes the DOM node directly. You no longer need to access the .element property.
    Affected components: Absolute, Block, Bottom, Column, Content, Flex, Grid, GridTile, Layout, Layout3D, List, Relative, Row, TabLink, Tab, Top, Transform, BackgroundImage, MaskImage.

✨ New Features

Section titled “✨ New Features”

  • New Navigation - Introduced a new utility component for handling spatial navigation, gamepad/keyboard interactions and focus management.

  • Feature Added Navigation.Area slot for grouping navigable components within the Navigation component.

  • Feature Custom Component Action Subscriptions: Custom components can now subscribe to Navigation actions.

🛠 Component Enhancements

Section titled “🛠 Component Enhancements”

Navigation Support

Section titled “Navigation Support”

The following components have been extended to support onAction and anchor props. They now respond to specific spatial navigation actions when placed inside a Navigation area:

ComponentPropsSupported Actions
Button, RoundedButtononAction anchor—
CheckboxPropsselect
AccordionPropsselect
DropdownPropsselect back
NumberInputPropsselect back move-up/down
PasswordInput, TextInputPropsselect back
Radio, Segment, Stepper, Slider, TextSliderPropsmove-left/right
XYSlider, ScrollPropspan
TooltipPropsback
TextBlock, InlineTextBlockonAction anchor—
ModalonAction anchor—
KeybindPropsselect
ColorPicker, RadialMenu, Carousel, TutorialonAction anchor—

Component improvements and fixes

Section titled “Component improvements and fixes”

  • New Toast: Added dismiss argument to the body function for programmatically closing a toast.

  • New Tutorial: Added onStart and onEnd props for adding a logic when the tutorial starts or ends.

  • New Modal: Forbid any mouse events outside the modal when there is a Modal.Overlay.

  • Fix Checkbox: Fixed an issue where the onChange handler was called on component mount.

  • Fix ToggleButton: Fixed checked prop not reacting to signal state changes.

  • Fix Slider & XYSlider: Fixed drag event not cleaning up if component unmounts during dragging.

📖 New Guides

Section titled “📖 New Guides”

  • New List - Provide a guide on how to initialize list items as navigable elements within a Navigation area.

  • New GridTile - Provide a guide on how to initialize grid tiles as navigable elements within a Navigation area.

  • New Keybinds - Provide an extended overview on gamepad support.

  • New ColorPicker - Provide a step by step guide on how to enable navigation support for the component.

  • New RadialMenu - Provide a step by step guide on how to enable navigation support for the component.

  • New Carousel - Provide a step by step guide on different ways to enable navigation for the component.

  • New Tutorial - Provide a step by step guide on different ways to enable navigation for the component.

  • New Toast - Provide a guide on how to close a toast programmatically.
